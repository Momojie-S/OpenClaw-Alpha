> 版本: v1.0
> 创建时间: 2026-03-03

## 核心原则

### 1. 来源信息

放在文档开头，必须包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| 更新日期 | 文档最后更新时间 | 2026-03-03 |
| 来源地址 | 官方文档或 API 地址 | https://tushare.pro/document/2?doc_id=259 |

### 2. 官方说明

**⚠️ 必须使用网络工具获取真实内容，禁止凭记忆或猜测编写！**

**允许的获取方式**：
- ✅ `web_fetch` 工具 - 访问官方文档 URL
- ✅ `curl` 命令 - 获取页面原始内容
- ❌ `web_search` 工具 - 搜索结果不是原始文档，禁止使用
- ❌ 凭记忆编写 - 容易出错，禁止使用
- ❌ 使用旧资料 - 信息可能过时，禁止使用

**获取后处理**：
1. 提取官方文档中的参数、字段说明
2. 保持原文，尽量不改动
3. 示例可精简，但参数和字段说明必须完整
4. 标注积分要求、单位等关键信息

### 3. 逐个更新原则

**⚠️ 每获取一个接口内容，立即更新一个文档！**

**禁止批量更新**：
- ❌ 获取多个接口内容后一次性批量更新
- ❌ 先获取所有内容再逐个写入

**正确流程**：
1. 使用 web_fetch 或 curl 获取接口 A 的内容
2. 立即更新接口 A 的文档文件
3. 使用 web_fetch 或 curl 获取接口 B 的内容
4. 立即更新接口 B 的文档文件
5. 依此类推...

**原因**：
- 大模型在批量处理时容易产生幻觉
- 逐个更新可以确保每个文档都基于真实内容
- 便于检查和校验每个文档的准确性

### 4. 额外说明

记录实际使用中发现的问题：
- 缺失字段
- 计算公式
- 注意事项（如积分要求、字段名差异等）

## 常见错误

### ❌ 错误示例 1：使用搜索结果

```
# 错误：使用 web_search 获取内容
web_search("tushare sw_daily 接口说明")
```

搜索结果是摘要，不是原始文档，可能不完整或有偏差。

### ❌ 错误示例 2：批量更新

```
# 错误：获取多个接口后批量更新
for api in apis:
    content = fetch(api.url)
    
for api, content in zip(apis, contents):
    write_doc(api, content)  # ❌ 此时内容可能已不准确
```

### ✅ 正确示例：逐个获取逐个更新

```
# 正确：获取一个，立即更新一个
content = web_fetch("https://tushare.pro/document/2?doc_id=327")
write_doc("sw_daily.md", content)  # ✅ 立即写入

content = web_fetch("https://tushare.pro/document/2?doc_id=181")
write_doc("index_classify.md", content)  # ✅ 立即写入
```
