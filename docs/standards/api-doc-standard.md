# 外部 API 文档规范

## 核心原则

### 1. 来源信息

文档开头必须包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| 更新日期 | 文档最后更新时间 | 2026-03-03 |
| 来源地址 | 官方文档或 API 地址 | https://tushare.pro/document/2?doc_id=259 |

### 2. 官方说明

**⚠️ 必须使用 web_fetch 或 curl 获取真实内容，禁止凭记忆或猜测编写！**

**获取后处理**：
1. 提取官方文档中的参数、字段说明
2. 保持原文，尽量不改动
3. 示例可精简，但参数和字段说明必须完整
4. 标注积分要求、单位等关键信息

### 3. 逐个更新原则

**⚠️ 每获取一个接口内容，立即更新一个文档！**

**正确流程**：
1. web_fetch 获取接口 A 内容 → 立即更新接口 A 文档
2. web_fetch 获取接口 B 内容 → 立即更新接口 B 文档
3. 依此类推...

**原因**：大模型在批量处理时容易产生幻觉，逐个更新可确保每个文档都基于真实内容。

## 检查清单

- [ ] 有更新日期和来源地址
- [ ] 官方说明来自 web_fetch 或 curl（不是 web_search）
- [ ] 官方说明保持原文
- [ ] 额外说明与官方说明分离
- [ ] 文件名使用接口函数名
- [ ] 积分要求、字段单位已标注

## 示例

**文件名**：`sw_daily.md`

```markdown
# 申万行业指数日线 - sw_daily

> 更新日期：2026-03-03
> 来源地址：https://tushare.pro/document/2?doc_id=259

---

## 官方说明

### 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| trade_date | str | 否 | 交易日期（YYYYMMDD） |
| start_date | str | 否 | 开始日期 |
| end_date | str | 否 | 结束日期 |
| index_code | str | 否 | 指数代码 |

### 字段

| 字段 | 说明 | 单位 |
|------|------|------|
| index_code | 指数代码 | - |
| name | 指数名称 | - |
| close | 收盘价 | - |
| pct_change | 涨跌幅 | % |

---

## 额外说明

### 注意事项

- 积分要求：120
- 字段名是 `pct_change`，不是 `pct_chg`
```
