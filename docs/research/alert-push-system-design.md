# 预警推送系统设计

> 研究日期：2026-03-08
> 研究目的：设计 OpenClaw-Alpha 的预警推送系统架构

---

## 需求分析

### 用户场景

1. **持仓风险预警**
   - 持仓股票出现业绩暴雷
   - 连续下跌触及止损线
   - 主力资金大幅流出

2. **自选股机会提醒**
   - 自选股出现买入信号
   - 技术指标突破
   - 估值回落到合理区间

3. **市场异动监控**
   - 北向资金大幅流入/流出
   - 板块热度快速上升
   - 涨停板异动

### 推送渠道

- 企业微信（已有）
- Discord（可选）
- 邮件（备选）

## 预警类型设计

### 1. 持仓风险预警

| 预警类型 | 触发条件 | 优先级 | 数据源 |
|---------|---------|:------:|--------|
| 业绩暴雷 | 业绩首亏/预减/增亏 | 高 | risk_alert |
| 连续下跌 | 连续 ≥3 天且累计跌幅 ≥10% | 高 | risk_alert |
| 资金流出 | 连续 ≥3 天净流出且累计 ≥1 亿 | 中 | risk_alert |
| 解禁压力 | 解禁占流通市值 ≥5% | 中 | restricted_release |
| 跌破止损 | 跌破用户设定止损线 | 高 | risk_alert |

### 2. 市场异动预警

| 预警类型 | 触发条件 | 优先级 | 数据源 |
|---------|---------|:------:|--------|
| 北向大幅流入 | 单日净流入 ≥50 亿 | 中 | northbound_flow |
| 北向大幅流出 | 单日净流出 ≥30 亿 | 高 | northbound_flow |
| 板块热度急升 | 热度环比 > +30% | 中 | industry_trend |
| 板块热度骤降 | 热度环比 < -30% | 中 | industry_trend |
| 涨停板异动 | 连板股数量异常 | 低 | limit_up_tracker |

### 3. 机会提醒（可选）

| 提醒类型 | 触发条件 | 优先级 | 数据源 |
|---------|---------|:------:|--------|
| 估值回落 | PE 回落到低估区间 | 低 | fundamental_analysis |
| 技术突破 | MACD 金叉/RSI 超卖反弹 | 低 | technical_indicators |
| 资金流入 | 连续大额流入 | 中 | stock_fund_flow |

## 技术实现方案

### 方案 A：Cron 定时任务（推荐）

**优点**：
- 精确时间控制
- 独立运行，不依赖主 session
- 可配置不同模型

**实现**：
```yaml
# OpenClaw cron 配置
jobs:
  - name: 早盘预警
    schedule: "0 9:15 * * 1-5"
    command: "检查持仓风险"
    
  - name: 收盘风险扫描
    schedule: "0 15:05 * * 1-5"
    command: "全面风险扫描并推送"
```

### 方案 B：Heartbeat 轮询

**优点**：
- 与主 session 集成
- 实现简单

**缺点**：
- 时间不精确
- 可能遗漏

**实现**：
- 在 HEARTBEAT.md 中添加检查项
- 主 agent 定期轮询

### 推荐：混合方案

- **关键预警**：使用 cron（收盘风险扫描）
- **日常检查**：使用 heartbeat（持仓监控）

## 配置设计

### 用户配置文件

```yaml
# ~/.openclaw/workspace-alpha/alert_config.yaml

# 监控标的
watchlist:
  - symbol: "000001"
    name: "平安银行"
    cost_price: 12.50  # 成本价（可选）
    stop_loss: 11.00   # 止损价（可选）
  - symbol: "600000"
    name: "浦发银行"

# 预警规则
rules:
  # 持仓风险
  risk_alert:
    enabled: true
    check_times: ["9:15", "15:05"]
    
  # 解禁风险
  restricted_release:
    enabled: true
    threshold: 5.0  # 占流通市值比例
    
  # 北向资金
  northbound_flow:
    enabled: true
    inflow_threshold: 50  # 亿
    outflow_threshold: 30  # 亿

# 推送渠道
channels:
  - type: wecom
    enabled: true
  - type: discord
    enabled: false
```

## 注意事项

1. **推送频率**：避免频繁打扰，关键预警才推送
2. **非交易时间**：周末和节假日不推送
3. **去重**：同一预警不重复推送
4. **静默时段**：夜间（23:00-8:00）不推送
