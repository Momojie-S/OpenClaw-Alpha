# 新闻分析系统设计

## 概述

两阶段新闻分析系统，从海量快讯中筛选价值内容，逐步深入。

```
海量新闻 → 快速分析（筛选层）→ 深度分析（精炼层）
              ↓                    ↓
         结构化 + 索引          深度洞察 + 建议
```

## 设计原则

### 调度与执行分离

```
┌─────────────────────────────────────────┐
│            调度层 (Backend)              │
│  - Backend Cron：定时拉取 + 触发智能体    │
│  - Backend API：手动触发智能体（调试）    │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│           执行层 (Agent Session)         │
│  - 智能体执行分析任务                     │
│  - 掌握 openclaw_alpha_* 全部 skill      │
│  - 可根据内容动态调整策略                 │
└─────────────────────────────────────────┘
```

**调度层职责**：
- 定时触发（每30分钟）
- 拉取新闻数据
- 创建 Agent Session 执行任务

**执行层职责**：
- 快速分析
- 热度计算
- 深度分析

### 两阶段漏斗

| 维度 | 快速分析 | 深度分析 |
|------|----------|----------|
| 处理量 | 全量新新闻 | 筛选后（<5%） |
| 目标 | 结构化 + 初判 | 深度洞察 + 建议 |
| 输出 | 标签 + 摘要 + 实体 + 热度 | 完整分析报告 |
| 触发 | 定时批量拉取 | 热度/关注触发 |
| 耗时 | 秒级/条 | 分钟级/条 |

### 快速分析的价值

**不是"简单分析"，而是"高效结构化"**：

1. **去重合并**：避免重复处理同一事件
2. **实体识别**：标注标的、板块，建立索引
3. **价值初判**：重要性、情绪、时效
4. **热度计算**：聚合传播、重要性、情绪、时效
5. **存储复用**：为深度分析和用户查询提供结构化数据

### 深度分析的价值

**不是"更多信息"，而是"更深洞察"**：

1. **事件解读**：发生了什么？为什么重要？
2. **影响评估**：对标的、板块、市场的影响
3. **数据整合**：结合资金流、公告、财报、技术面
4. **操作建议**：应该关注什么？风险在哪？

## 模块划分

### 1. 数据采集（Backend）

**职责**：Backend Cron/API 触发，从 RSSHub 拉取新闻

**触发方式**：
- Backend Cron：每30分钟自动执行
- Backend API：手动触发（调试）

**输出**：新闻列表 → 传递给 Agent Session

**关键问题**：
- 如何判定"新"？（已处理状态 vs 发布时间）
- 如何快速去重？（标题相似度 + 时间窗口）

**设计文档**：[data-collection.md](data-collection.md)（待编写）

### 2. 快速分析（Agent Session）

**职责**：Agent Session 内执行，单条新闻的快速分析

**设计原则**：
- **不限制取值**：不预设固定的事件类型、影响范围、确定性等字段取值
- **流程引导**：引导智能体分析关键要素（板块、公司、影响程度）
- **自主判断**：让智能体自己决定是否值得深入分析
- **完整识别**：包括未上市公司的识别

**输入**：单条新闻（标题 + 正文 + 来源 + 时间）
**输出**：`analysis.json`（简单结构）

```json
{
  "related_sectors": ["板块1", "板块2"],
  "related_companies": [
    {"name": "公司名", "listed": true, "code": "000001"},
    {"name": "未上市公司名", "listed": false, "code": null}
  ],
  "impact_assessment": "自然语言描述：这件事对板块/公司有什么影响，影响程度如何",
  "worth_deep_analysis": true,
  "reason": "理由：为什么值得/不值得深入分析"
}
```

**分析流程**：
1. **相关板块**：哪些板块会受到影响？
2. **相关公司**：哪些公司会受到影响？（包括未上市）
3. **影响程度**：影响有多大？是正面还是负面？
4. **是否值得深入分析**：基于以上分析，判断是否需要深入

**Backend 处理**：读取 `worth_deep_analysis` 字段，决定是否触发深度分析

**关键问题**：
- 相关公司识别：如何提高识别未上市公司的准确性？
- 影响程度描述：是否需要更结构化？
- Prompt 设计：如何引导智能体按流程分析？

**设计文档**：[quick-analysis.md](quick-analysis.md)

### 3. 热度计算（Agent Session）

**职责**：Agent Session 内执行，计算新闻、标的、板块热度，维护榜单

**输入**：快速分析结果（批量）
**输出**：热度榜单

**热度模型**：见 [news-hotness.md](../news-hotness.md)

**关键问题**：
- 多源重复如何判定？
- 标的/板块热度如何聚合？
- 榜单更新频率？

**设计文档**：[hotness-ranking.md](hotness-ranking.md)（待编写）

### 4. 深度分析（Agent Session）

**职责**：Backend 触发独立的 Agent Session，对重要新闻进行深度分析

**触发条件**（Backend Python 代码判断）：
- event_type = 政策/监管/并购
- scope = 全市场
- amount ≥ 100亿
- emotion_score ≥ 8 且 certainty = 确认

**输入**：新闻 + 快速分析结果 (`analysis.json`)
**输出**：`report.md`（深度分析报告）

```python
{
    "event_summary": "事件概述：央行降准0.5个百分点，释放长期资金约1万亿元",
    "why_important": "为什么重要：这是2025年以来的首次降准，释放的1万亿资金将显著降低市场资金成本，提振市场信心",
    "impact": {
        "market": "对市场的影响：利好银行、地产等资金密集型板块，可能带动指数上行",
        "sectors": ["银行", "地产", "券商"],
        "stocks": ["000001", "600000", "600036"]
    },
    "data_integration": {
        "fund_flow": "资金流向分析（调用 fund_flow skill）",
        "related_news": "相关历史新闻（最近3个月降准相关）",
        "announcements": "相关公告（银行股近期公告）"
    },
    "suggestions": [
        "关注银行板块龙头股",
        "注意地产股风险（政策效果需观察）",
        "券商可能受益于成交量放大"
    ]
}
```

**关键问题**：
- Skill 调用：如何调用其他 skill 获取辅助数据？
- 报告格式：如何让报告有价值且易读？
- 深度控制：如何避免过度分析？

**设计文档**：[deep-analysis.md](deep-analysis.md)（待编写）

### 5. 存储与索引

**职责**：持久化分析结果，支持查询

**目录结构**：
```
workspace/news/
├── cache/
│   └── rsshub/
│       └── {route_id}/
│           └── {YYYY-MM-DD}.json    # 原始新闻 + 处理状态
├── quick_analysis/
│   └── {YYYY-MM-DD}/
│       └── {news_id}.json           # 快速分析结果
├── hotness/
│   ├── news_rank.json               # 新闻热度榜
│   ├── stock_rank.json              # 标的热度榜
│   └── sector_rank.json             # 板块热度榜
└── deep_analysis/
    └── {news_id}/
        ├── progress.md
        └── report.md
```

**关键问题**：
- 查询效率：如何快速找到"某股票相关的新闻"？
- 索引设计：是否需要额外的索引文件？
- 清理策略：历史数据如何清理？

**设计文档**：[storage.md](storage.md)（待编写）

### 6. 触发与队列

**职责**：管理深度分析的触发条件

**触发条件**：
1. **自动触发**：
   - hotness ≥ 8.0
   - hotness ≥ 6.0 且持续 3 小时
   - 事件类型 = 政策/监管/重大事件
2. **人工触发**：
   - 用户关注标的/板块相关
   - 用户自选股相关

**队列管理**：
- 优先级队列（按热度排序）
- 去重（24小时内同一新闻不重复分析）
- 并发控制（同时最多 N 个深度分析）

**设计文档**：[trigger-queue.md](trigger-queue.md)（待编写）
