# 新闻热度分析设计文档

## 概述

新闻热度分析系统用于从快讯流中识别热门新闻、标的和板块，并自动触发深度分析。核心原则：**纯新闻判断，不依赖市场数据**。

## 核心目标

1. **多维热度模型**：传播(40%) + 重要性(30%) + 情绪(20%) + 时效(10%)
2. **三级榜单**：新闻榜 / 标的榜 / 板块榜
3. **智能触发**：自动识别需要深度分析的内容

---

## 1. 热度模型

### 1.1 热度分数公式

```
热度总分 = 传播热度(40%) + 重要性热度(30%) + 情绪热度(20%) + 时效热度(10%)
```

### 1.2 四大维度

| 维度 | 权重 | 子指标 |
|------|------|--------|
| 传播热度 | 40% | 多源重复数、相似度聚类大小、突发密集度 |
| 重要性热度 | 30% | 事件类型权重、影响范围、金额规模 |
| 情绪热度 | 20% | 情绪词强度、确定性词汇 |
| 时效热度 | 10% | 新鲜度(发布时间)、更新频率 |

---

## 2. 传播热度

### 2.1 多源重复判断

**问题**：如何判断两条新闻是否报道"同一事件"？

**相似度计算**：

```
similarity = 0.5 × title_similarity + 0.3 × content_similarity + 0.2 × entity_overlap
```

- **标题相似度**：编辑距离 / 余弦相似度
- **内容相似度**：TF-IDF + Cosine / SimHash
- **实体重叠**：抽取的标的/板块/人名/地名 Jaccard 相似度

**阈值设计**：

| 相似度范围 | 判定 | 处理方式 |
|------------|------|----------|
| ≥ 0.85 | 完全重复 | 去重，计入重复数 |
| 0.6 - 0.85 | 同一事件 | 聚类，计入重复数 |
| < 0.6 | 不同事件 | 独立处理 |

**时间窗口**：同一事件判定只在 ±4 小时内进行

### 2.2 突发密集度

```
burst_score = count(news in last_1h) / baseline_avg
```

- baseline_avg：过去 7 天同一小时的平均新闻数
- burst_score > 3：判定为突发

## 3. 重要性热度

### 3.1 事件类型权重

| 事件类型 | 权重 | 示例 |
|----------|------|------|
| 政策/监管 | 10 | 降息、新规、禁令 |
| 重大事件 | 8 | 并购、重组、破产 |
| 业绩公告 | 6 | 财报、预告、快报 |
| 行业动态 | 5 | 技术突破、产能变化 |
| 市场数据 | 3 | 涨跌、成交量 |
| 一般资讯 | 2 | 常规报道 |

### 3.2 影响范围

| 范围 | 分数 |
|------|------|
| 全市场/宏观经济 | 10 |
| 多行业/跨板块 | 7 |
| 单行业/板块 | 5 |
| 单一标的 | 3 |

### 3.3 金额规模

| 金额区间 | 分数 |
|----------|------|
| ≥ 100亿 | 10 |
| 10亿 - 100亿 | 7 |
| 1亿 - 10亿 | 5 |
| < 1亿 | 2 |
| 无金额信息 | 5 (默认) |

## 4. 情绪热度

### 4.1 情绪词库

**极度情绪词** (score = 10)：
- 暴涨、暴跌、涨停、跌停、熔断、闪崩

**强情绪词** (score = 7)：
- 大涨、大跌、飙升、重挫、狂泻、井喷

**中等情绪词** (score = 5)：
- 上涨、下跌、走高、走低、回调、反弹

**弱情绪词** (score = 3)：
- 小涨、小跌、微跌、微涨

### 4.2 确定性词汇

| 类型 | 示例 | 加成 |
|------|------|------|
| 确认发生 | 已完成、正式、官宣 | +2 |
| 高确定性 | 确定、将、预计 | +1 |
| 不确定 | 或许、可能、传闻 | -1 |
| 谣言 | 网传、未经证实 | -3 |

### 4.3 情绪热度计算

```
emotion_score = max(情绪词分数) + 确定性加成
emotion_score = clamp(emotion_score, 0, 10)
```

## 5. 时效热度

### 5.1 新鲜度

| 发布时间 | 分数 |
|----------|------|
| < 30分钟 | 10 |
| 30分钟 - 1小时 | 8 |
| 1小时 - 2小时 | 6 |
| 2小时 - 4小时 | 4 |
| 4小时 - 8小时 | 2 |
| > 8小时 | 1 |

### 5.2 更新频率

- 同一事件有后续报道：+2 (最多 +4)
- 表示事件持续发酵

## 6. 实体识别与聚合

### 6.1 实体类型

| 实体类型 | 识别方式 |
|----------|----------|
| 股票代码 | 正则匹配 (6位数字) |
| 股票名称 | 词典匹配 |
| 板块/概念 | 词典匹配 + 规则 |
| 人物 | NER + 词典 |
| 机构 | NER + 词典 |

### 6.2 标的热度聚合

```
stock_hotness = Σ(news_hotness × relevance) / time_decay_factor
```

- relevance：新闻与标的的相关度 (0-1)
- time_decay_factor：时间衰减 (1小时内=1, 4小时内=0.7, 12小时内=0.4)

### 6.3 板块热度聚合

```
sector_hotness = Σ(stock_hotness in sector) × member_count_factor
```

- member_count_factor：1 / log(member_count + 1)，避免大板块天然高分

## 8. 深度分析触发

### 8.1 触发条件

| 条件 | 说明 |
|------|------|
| hotness ≥ 8.0 | 热度阈值触发 |
| hotness ≥ 6.0 持续 3小时 | 持续热点触发 |
| 事件类型 = 政策/监管/重大事件 | 重要事件强制触发 |

### 8.2 队列管理

```python
class AnalysisQueue:
    def __init__(self):
        self.queue = PriorityQueue()  # 按 hotness 排序
        self.processing = set()       # 正在处理的
        self.completed = set()        # 已完成的 (24小时内去重)

    def add(self, news_id, hotness_score):
        if news_id in self.completed:
            return
        self.queue.put((-hotness_score, news_id))  # 负数实现大顶堆
```

### 8.3 优先级策略

1. 热度分数高的优先
2. 同等热度下，政策/监管事件优先
3. 同等热度下，新鲜度高的优先
