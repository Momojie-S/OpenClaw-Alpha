## 一、背景与目标

### 1.1 业务背景

个人投资者需要快速了解 A 股市场热度，识别：
- 当前市场热点板块
- 领涨/领跌板块
- 热门概念题材
- 热度变化趋势

### 1.2 业务目标

提供市场热度分析能力，帮助用户：
- 快速了解市场整体热度分布
- 识别强势板块和弱势板块
- 追踪热度变化趋势

### 1.3 目标用户

- 个人投资者（Momojie）
- 需要快速了解市场热度的用户

## 二、用户场景

### 2.1 核心场景

## 三、业务概念

### 3.1 核心概念定义

| 概念 | 定义 |
|------|------|
| 板块 | 行业板块或概念板块的统称 |
| 热度指数 | 综合反映板块市场关注度的指标（0-100） |
| 趋势信号 | 热度变化方向的判断（加热中/降温中/稳定） |
| 环比 | 与上一交易日对比的变化幅度 |

### 3.2 热度指数

热度指数反映板块的市场关注度，由以下维度加权计算：

| 维度 | 权重 | 说明 |
|------|------|------|
| 涨跌幅 | 30% | 价格变动幅度 |
| 换手率 | 25% | 交易活跃程度 |
| 资金净流入 | 25% | 主力资金方向 |
| 涨跌家数比 | 20% | 板块内股票表现 |

### 3.3 趋势信号

| 信号 | 判断规则 |
|------|----------|
| 加热中 | 热度环比 > +20% 且 涨幅 > 0 且 资金流入 > 0 |
| 降温中 | 热度环比 < -20% 或 (涨幅 < -3% 且 资金流出) |
| 稳定 | 其他情况 |

### 3.4 换手率

换手率 = 成交额 / 流通市值 × 100%

## 四、验收标准

### 4.1 功能验收

- [ ] 能输出板块热度排名（TOP N）
- [ ] 热度指数计算符合上述公式
- [ ] 趋势信号判断符合上述规则
- [ ] 能对比上一交易日（环比）
- [ ] 支持行业板块和概念板块
- [ ] 输出格式为结构化数据（JSON）

### 4.2 非功能验收

- [ ] 响应时间 < 5 秒
- [ ] 非交易日有明确提示

## 五、范围与优先级

### 5.1 本期范围

- 申万行业板块热度
- 概念板块热度
- 热度排名
- 趋势信号

### 5.2 后续范围

- 个股热度排名
- 板块成分股
- 资金流向详情
- 历史热度曲线

### 5.3 不做

- 实时推送
- 复杂的量化策略
- 自动交易

## 六、数据需求

### 6.1 板块数据字段

| 字段 | 说明 | 用途 |
|------|------|------|
| 板块代码 | 唯一标识 | 标识 |
| 板块名称 | 显示名称 | 显示 |
| 涨跌幅 | 当日涨跌幅 | 热度计算 |
| 换手率 | 交易活跃度 | 热度计算 |
| 资金净流入 | 主力资金方向 | 热度计算 |
| 涨跌家数 | 板块内股票表现 | 热度计算 |

### 6.2 历史数据

需要存储历史数据以支持环比分析。
