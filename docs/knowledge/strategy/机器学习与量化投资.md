# 机器学习与量化投资

> ⚠️ **技术约束**：算力有限，不涉及机器学习相关开发。本笔记仅供了解原理，不进行 API 调研或实际开发。

## 定义

将机器学习算法应用于量化投资，用于因子挖掘、非线性因子组合、收益预测和风险管理。相比传统线性模型，ML 能捕捉更复杂的市场规律。

---

## 应用场景

| 场景 | 传统方法 | ML 方法 |
|------|----------|---------|
| 因子挖掘 | 人工设计、文献 | 自动发现非线性模式 |
| 因子组合 | 线性加权（IC/IR） | 非线性组合、自动学习权重 |
| 收益预测 | 线性回归 | 树模型、神经网络 |
| 风险估计 | 历史协方差 | 条件协方差、尾部风险建模 |

---

## 常用模型

### 树模型（主流）

**LightGBM / XGBoost / CatBoost**

**优点**：
- 处理非线性关系能力强
- 自动处理缺失值
- 特征重要性可解释
- 训练速度快

**典型流程**：
1. 构建因子池（100-500 个特征）
2. 特征工程（去极值、标准化、行业中性化）
3. 训练模型（预测下期收益或分类）
4. 生成信号 / 排序

**超参数**：
- 树深度（3-6）
- 学习率（0.01-0.1）
- 叶节点数（31-127）
- 正则化（L1/L2）

### 神经网络

**MLP（多层感知机）**
- 适合因子组合
- 需要大量数据

**LSTM / GRU**
- 捕捉时间序列依赖
- 适合高频数据

**Transformer**
- 并行处理序列
- 注意力机制可解释

**缺点**：
- 容易过拟合
- 需要大量数据
- 调参复杂
- 黑盒难解释

### 集成方法

**Stacking**
- 多个基模型预测作为特征
- 元模型学习最优组合

**典型组合**：
- LightGBM + XGBoost + CatBoost
- 树模型 + 线性模型

## 特征工程

### 因子预处理

```
原始因子 → 去极值(MAD) → 标准化(Z-score) → 行业中性化
```

### 时间序列特征

| 特征类型 | 示例 |
|---------|------|
| 滚动统计 | 均值、标准差、偏度、峰度 |
| 滚动排名 | 过去 N 期分位数 |
| 趋势特征 | 线性斜率、动量 |
| 波动特征 | 波动率、ATR |

### 交互项

**手动构造**：
- Size × Value
- Momentum × Volatility

**自动学习**：
- 树模型自动捕捉交互
- FM/DeepFM 显式建模

## 模型训练

### 数据划分

**时间序列划分**（Walk-Forward）：
```
Train: 2015-2018 | Valid: 2019 | Test: 2020
Train: 2016-2019 | Valid: 2020 | Test: 2021
```

**禁止**：随机划分（未来泄露）

### 目标函数

**回归**：
- 预测下期收益
- MSE / MAE 损失

**分类**：
- 预测涨跌 / 分组
- 交叉熵损失

**排序**：
- 预测相对排名
- LambdaRank / ListNet

### 超参数调优

**方法**：
- Grid Search
- Random Search
- Bayesian Optimization（Optuna）

**注意**：
- 在验证集上调参，非测试集
- 避免过度调参导致过拟合

---

## 过拟合防控

### 核心原则

| 方法 | 说明 |
|------|------|
| 时间序列划分 | 禁止随机划分 |
| 样本外测试 | 保留真正的测试集 |
| 正则化 | L1/L2、Dropout、Early Stopping |
| 特征筛选 | 剔除噪声因子 |
| 简单模型优先 | 先用线性模型建立 baseline |

### 信号衰减检验

检验模型信号是否随时间衰减：
- 训练集 IC vs 测试集 IC
- 若衰减 > 50%，可能过拟合

### Purging & Embargo

**Purging**：移除与测试期重叠的训练样本
**Embargo**：在训练-测试之间设置缓冲期

## 可解释性

### 特征重要性

**树模型**：
- Gain：分裂带来的信息增益
- Split：被用于分裂的次数
- SHAP：Shapley 值，更准确

**SHAP 分析**：
- 全局重要性排序
- 单样本解释
- 因子方向分析

### 部分依赖图（PDP）

展示单个因子与预测的关系，控制其他因子不变。

---

## 实践建议

### 渐进式应用

1. **Baseline**：线性模型（IC 加权）
2. **简单 ML**：单树模型（LightGBM）
3. **复杂 ML**：集成、神经网络
4. **对比验证**：ML vs 线性，是否显著提升

### 不要迷信 ML

研究表明（Alpha Architect, 2024）：
- ML 对因子收益的改进**有限**
- 线性模型在很多情况下已经足够
- ML 的价值更多在于发现新因子，而非因子组合

### 市场制度变化

ML 模型在市场制度切换时可能失效：
- 需要持续监控模型表现
- 设置止损 / 回撤控制
- 定期重新训练

---

## 挑战

| 挑战 | 应对 |
|------|------|
| 过拟合 | 时间序列划分、正则化、简单优先 |
| 数据泄露 | Purging、Embargo、严格的时间控制 |
| 可解释性 | SHAP、特征重要性、PDP |
| 计算成本 | 云端训练、分布式计算 |
| 市场变化 | 持续监控、定期重训练 |
| 信号拥挤 | 独特因子、另类数据 |
