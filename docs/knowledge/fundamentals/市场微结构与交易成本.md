# 市场微结构与交易成本

## 定义

市场微结构研究交易机制如何影响价格形成、流动性、交易成本。理解这些概念有助于优化执行、降低交易成本、提升实际收益。

---

## 交易成本构成

### 显性成本

**手续费**：
- 佣金：券商收取（通常万 2-万 3）
- 印花税：卖出时 0.1%（A 股）
- 过户费：万分之 0.1（A 股）

**计算**：
```
总成本 = 佣金 × 2 + 印花税 + 过户费
       ≈ 0.03% + 0.1% + 0.001%
       ≈ 0.13% (单边)
```

### 隐性成本

| 成本类型 | 定义 | 影响因素 |
|---------|------|---------|
| 价差成本 | 买卖价差 | 流动性、波动率 |
| 冲击成本 | 交易导致价格不利变动 | 订单大小、流动性 |
| 时机成本 | 等待期间价格变化 | 波动率、等待时间 |
| 机会成本 | 无法完成交易的损失 | 流动性不足 |

**隐性成本估算**：
```
隐性成本 ≈ 1/2 × 价差 + 冲击成本 + 时机成本
```

## 买卖价差

### 定义

**买卖价差 (Bid-Ask Spread)**：最优买价与最优卖价的差额。

$$Spread = Ask - Bid$$

**相对价差**：
$$Relative\ Spread = \frac{Ask - Bid}{Mid}$$

### 价差构成

| 组成 | 含义 | 特点 |
|------|------|------|
| 订单处理成本 | 做市商运营成本 | 相对固定 |
| 库存成本 | 持有风险头寸的补偿 | 随波动率变化 |
| 逆向选择成本 | 与知情交易者交易的损失 | 信息不对称时高 |

### 价差影响因素

| 因素 | 影响 |
|------|------|
| 流动性 | 流动性好 → 价差小 |
| 波动率 | 波动大 → 价差大 |
| 交易量 | 交易活跃 → 价差小 |
| 信息事件 | 财报/新闻前 → 价差扩大 |
| 时段 | 开盘/收盘 → 价差较大 |

### A 股价差特点

- 散户为主，价差相对较小
- 涨跌停时流动性丧失
- 小盘股价差大
- 集合竞价阶段价差波动大

## 市场冲击

### 定义

**市场冲击 (Market Impact)**：大额交易导致价格向不利方向变动。

**买入冲击**：推高价格
**卖出冲击**：压低价格

### 冲击模型

**线性模型**（简单估算）：
$$Impact = \sigma \times \frac{Volume}{ADV} \times \theta$$

其中：
- σ = 日波动率
- Volume = 交易量
- ADV = 平均日成交额
- θ = 冲击系数（通常 0.1-0.3）

**平方根模型**（更常用）：
$$Impact = \sigma \times \sqrt{\frac{Volume}{ADV}} \times \theta$$

### 冲击成本估算

| 交易量占 ADV | 估计冲击成本 |
|-------------|------------|
| 1% | < 0.1% |
| 5% | 0.2-0.3% |
| 10% | 0.4-0.6% |
| 20% | 0.8-1.2% |
| 50% | 2-3% |

### 降低冲击策略

1. **分批交易**：大单拆成小单
2. **延长执行时间**：降低单时点交易量
3. **选择流动性好的标的**
4. **避开信息敏感时点**

## 流动性度量

### 宽度（Width）

**定义**：交易成本的高低

**指标**：
- 买卖价差
- 相对价差

### 深度（Depth）

**定义**：不影响价格的最大交易量

**指标**：
- 限价订单簿深度（各价位挂单量）
- 换手率
- 成交额

### 即时性（Immediacy）

**定义**：完成交易的速度

**指标**：
- 成交时间
- 订单执行率

### 弹性（Resiliency）

**定义**：价格受冲击后恢复的速度

**指标**：
- 价格恢复时间
- 订单流不平衡恢复

### 综合指标

**Amihud 非流动性**：
$$ILLIQ = \frac{|R_t|}{Volume_t}$$

含义：单位成交额带来的价格变动

**Kyle's Lambda**：
$$\lambda = \frac{\Delta P}{Order\ Flow}$$

含义：单位订单流引起的价格变化

## 执行算法

### TWAP（时间加权平均价格）

**策略**：将订单均匀分布在时间段内执行

**适用**：
- 流动性较好的标的
- 无明显趋势时
- 降低冲击成本

**缺点**：
- 不考虑市场状态
- 可能错过流动性高峰

### VWAP（成交量加权平均价格）

**策略**：按历史成交量分布执行

**步骤**：
1. 获取历史成交量分布曲线
2. 按分布分配订单
3. 实时监控执行

**适用**：
- 追踪市场平均价格
- 流动性规律明显

**缺点**：
- 依赖历史规律
- 特殊事件时失效

### POV（参与率策略）

**策略**：按固定比例参与市场成交

$$Target\ Volume = Participation\ Rate \times Market\ Volume$$

**适用**：
- 控制市场参与度
- 大额订单执行

**风险**：
- 市场成交骤减时执行缓慢

### IS（Implementation Shortfall）

**策略**：最小化执行偏差

**目标函数**：
$$\min IS = |Execution\ Price - Decision\ Price|$$

**特点**：
- 考虑时机成本
- 平衡冲击与时机风险

## 订单类型

### 限价单（Limit Order）

**定义**：指定价格买入/卖出

**优点**：
- 控制价格
- 提供流动性

**风险**：
- 可能不成交
- 逆向选择（被知情交易者"狙击"）

### 市价单（Market Order）

**定义**：以当前最优价格立即成交

**优点**：
- 即时成交
- 确定性好

**风险**：
- 价格不确定
- 支付价差成本
- 大单可能"穿透"多档

### 算法单

| 类型 | 用途 |
|------|------|
| Iceberg（冰山单） | 隐藏真实订单大小 |
| Stop（止损单） | 触发价后执行 |
| Fill-or-Kill（FOK） | 全部成交或取消 |
| Immediate-or-Cancel（IOC） | 立即成交或取消 |

## 交易成本分析（TCA）

### 核心指标

**Implementation Shortfall（IS）**：
$$IS = Decision\ Price - Execution\ Price$$

**分解**：
```
IS = 择时成本 + 冲击成本 + 机会成本
```

| 成本 | 公式 |
|------|------|
| 择时成本 | (Arrival Price - Decision Price) × Quantity |
| 冲击成本 | (Execution Price - Arrival Price) × Quantity |
| 机会成本 | (Decision Price - End Price) × Unfilled |

### 基准价格

| 基准 | 定义 | 用途 |
|------|------|------|
| 决策价 | 下单时刻价格 | 评估总体执行 |
| 到达价 | 订单到达市场时价格 | 评估冲击成本 |
| VWAP | 成交量加权均价 | 对比市场平均 |
| 收盘价 | 当日收盘价 | 长期评估 |

### 分析维度

1. **按标的**：哪些股票执行成本高？
2. **按时段**：何时执行效果最好？
3. **按订单大小**：冲击成本与订单量的关系
4. **按策略**：不同执行策略的对比

## 实践建议

### 降低成本的策略

| 策略 | 说明 | 节省成本 |
|------|------|---------|
| 分批交易 | 降低单笔冲击 | 0.1-0.3% |
| 选择时段 | 开盘后 30 分钟、收盘前 | 0.05-0.1% |
| 限价单为主 | 减少价差支付 | 0.05-0.15% |
| 避开事件 | 财报、新闻前后 | 0.1-0.5% |
| 控制换手 | 减少不必要的交易 | 累计显著 |

### 不同规模的策略

| 资金规模 | 策略重点 |
|---------|---------|
| 小资金（<100 万） | 关注佣金、简单执行 |
| 中等资金（100-1000 万） | 关注价差、适度分批 |
| 大资金（>1000 万） | 关注冲击成本、算法执行 |

### A 股特殊考量

1. **涨跌停风险**：流动性完全丧失
2. **集合竞价**：开盘/收盘的执行策略
3. **龙虎榜**：大额交易信息披露
4. **T+1**：日内无法纠错
