# 风险价值

## 定义

风险价值（Value at Risk, VaR）是在给定置信水平和持有期内，投资组合可能遭受的最大损失。

**通俗理解**：95% 的 VaR = 100 万，意味着有 95% 的把握损失不会超过 100 万。

## 计算方法

### 1. 参数法（方差-协方差法）

**假设**：收益率服从正态分布

```python
VaR = μ - σ × z × √t
```

- μ：预期收益率
- σ：波动率
- z：置信水平对应的 Z 值（95% = 1.645，99% = 2.326）
- t：持有期（天）

**优点**：计算简单
**缺点**：正态分布假设不成立，低估尾部风险

### 2. 历史模拟法

**步骤**：
1. 收集历史收益率数据
2. 排序后取分位数

**优点**：无需分布假设
**缺点**：依赖历史数据，无法预测新风险

### 3. 蒙特卡洛模拟

**步骤**：
1. 建立收益率模型
2. 模拟大量路径
3. 统计损失分布

**优点**：灵活，可处理复杂分布
**缺点**：计算量大，模型依赖性强

## 持有期和置信水平

| 置信水平 | Z 值 | 适用场景 |
|---------|------|----------|
| 90% | 1.282 | 内部风险管理 |
| 95% | 1.645 | 巴塞尔协议标准 |
| 99% | 2.326 | 极端风险控制 |

**持有期选择**：
- 1 天：日内交易
- 10 天：巴塞尔协议要求
- 1 个月：中长期投资

## 应用场景

### 1. 风险限额

```
单只股票 VaR < 组合 VaR × 10%
单一行业 VaR < 组合 VaR × 30%
```

### 2. 资本配置

- 银行：监管资本 = VaR × 3
- 对冲基金：根据 VaR 调整杠杆

### 3. 绩效评估

- 风险调整收益 = 收益 / VaR
- 比较不同策略的风险效率

## 局限性

### 1. 尾部风险低估

- VaR 只告诉"最大损失是多少"，不告诉"超过最大损失后会亏多少"
- 解决：使用 CVaR（条件风险价值）

### 2. 非正态分布

- 股票收益率有肥尾特征
- 解决：使用 t 分布或历史模拟

### 3. 相关性变化

- 危机时相关性飙升，分散化失效
- 解决：压力测试 + Copula 模型

### 4. 流动性风险

- VaR 假设可以立即平仓
- 解决：调整持有期或加入流动性折扣

## CVaR（条件风险价值）

**定义**：超过 VaR 时的平均损失

```python
CVaR = E[损失 | 损失 > VaR]
```

**优势**：
- 捕捉尾部风险
- 满足次可加性（组合 CVaR ≤ 单个 CVaR 之和）

**A股应用**：
- 涨跌停限制 → 流动性风险高
- 建议：VaR + CVaR + 压力测试组合使用

## A 股特点

### 1. 波动率高

- A 股年化波动率 25-35%，美股 15-20%
- VaR 计算需更高置信水平

### 2. 涨跌停限制

- 流动性风险被低估
- 建议持有期 ≥ 3 天

### 3. 相关性高

- 系统性风险主导
- 分散化效果有限

## 实践建议

### 1. 方法选择

| 场景 | 推荐方法 |
|------|----------|
| 快速估算 | 参数法 |
| 历史重现 | 历史模拟 |
| 复杂组合 | 蒙特卡洛 |

### 2. 参数设置

- 置信水平：95% 或 99%
- 持有期：与实际持仓周期匹配
- 回溯期：1-3 年

### 3. 风险管理流程

```
计算 VaR → 与限额对比 → 超限调整仓位 → 定期回测验证
```
