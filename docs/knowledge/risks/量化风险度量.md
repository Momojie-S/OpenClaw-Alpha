# 量化风险度量

## 定义

用量化方法衡量投资组合潜在损失的指标体系，回答"在多大程度上可能损失多少"的问题。

---

## VaR（风险价值）

### 定义

**风险价值 (VaR, Value at Risk)**：在给定置信水平和持有期内，投资组合可能遭受的最大潜在损失。

**通俗解释**：在正常市场条件下，有 X% 的把握说，在未来 N 天内，损失不会超过 Y 金额。

**示例**：95% 置信度、1 天持有期的 VaR 为 100 万，意味着有 95% 的概率单日损失不超过 100 万。

### 计算方法

#### 1. 参数法（方差-协方差法）

**假设**：收益率服从正态分布

**公式**：
```
VaR = 投资组合市值 × σ × Z
```
- σ：收益率标准差
- Z：置信水平对应的 Z 值（95%→1.65，99%→2.33）

**优点**：计算简单、速度快

**缺点**：
- 假设正态分布，忽略"厚尾"现象
- 对期权等非线性资产不适用

---

#### 2. 历史模拟法

**假设**：历史会重演

**步骤**：
1. 收集历史收益率数据（建议 250 天以上）
2. 对当前组合应用历史变化，计算每日损益
3. 排序损益，取对应分位点

**优点**：
- 无分布假设
- 能捕捉"厚尾"和非线性

**缺点**：
- 依赖历史数据窗口
- 极端事件样本不足时低估风险

---

#### 3. 蒙特卡洛模拟法

**假设**：风险因子服从特定分布

**步骤**：
1. 定义风险因子及分布
2. 随机生成数万个情景
3. 对每个情景重新估值组合
4. 排序损益，取分位点

**优点**：
- 最灵活、最准确
- 适用于复杂组合

**缺点**：
- 计算量大
- 模型风险高

---

### 方法对比

| 方法 | 计算速度 | 准确性 | 适用场景 |
|------|---------|--------|---------|
| 参数法 | 快 | 中 | 简单组合 |
| 历史模拟法 | 中 | 中 | 一般组合 |
| 蒙特卡洛 | 慢 | 高 | 复杂组合（含期权） |

---

### VaR 的局限性

1. **不告诉损失超过 VaR 会怎样**：只说"不超过"，不说"超过后平均损失多少"
2. **低估极端风险**：正态分布假设下，极端事件概率被低估
3. **非次可加**：组合 VaR 可能大于各资产 VaR 之和，违背分散化原则

---

### VaR 的历史背景

**起源（1987 年黑色星期一后）**

- **1987 股灾**：标准统计模型无法预测如此极端的市场崩盘，引发对量化风险管理的反思
- **早期发展**：Bankers Trust 等机构的量化交易团队在 1990 年前已建立 VaR 系统，但未标准化
- **RiskMetrics（1994）**：J.P. Morgan 首次公开 VaR 方法论，并提供免费参数估计，推动行业标准化
- **监管采纳**：
  - 1997 年 SEC 要求上市公司披露衍生品风险，银行选择用 VaR 满足要求
  - 1999 年起巴塞尔 II 协议采纳 VaR 作为市场风险度量标准

**关键转折点**

> J.P. Morgan CEO Dennis Weatherstone 著名的要求："下午 4:15 之前给我一份将全公司风险汇总到一页纸上的报告。"

这一需求推动了 VaR 从交易台工具进化为企业级风险汇总指标。

---

### VaR 的争议与批评

**Nassim Taleb 的批评（1997-2009）**

1. **忽视历史经验**：用未经实战检验的模型取代 2500 年的市场智慧
2. **江湖骗术**：声称能估计极端事件风险，这在统计学上是不可能的
3. **虚假安全感**：给管理层和监管者错误的安全感
4. **被交易员利用**：交易员会针对 VaR 模型的盲点进行套利

**David Einhorn 的比喻（2008）**

> "VaR 就像一个安全气囊，在所有时候都工作正常，除了车祸发生时。"

**学术界的批评**

- **尾部风险不可测**：极端事件样本不足，无法可靠估计其概率
- **锚定效应**：VaR 作为单一数字容易导致过度依赖，忽视其他风险信号
- **非连贯性**：不满足次可加性，分散化反而可能增加 VaR

**2008 金融危机的教训**

许多机构在危机前 VaR 看起来很低，但仍然遭受巨额损失：
- VaR 模型基于历史数据，未考虑危机时的相关性变化
- 低估了流动性风险（无法在跌停时卖出）
- 忽视了连锁反应（交易对手违约、市场信心崩溃）

---

### 三区管理框架

VaR 的一个实用价值是将风险划分为三个管理区域：

**区域 1：1-3 倍 VaR（正常波动）**

- **特征**：日常市场波动，VaR 突破是预期事件
- **管理方式**：
  - 用统计方法分析
  - 短期数据足够可靠
  - 概率估计有意义
- **心态**：这是做生意的一部分，无需过度担心

**区域 2：3-10 倍 VaR（压力测试区）**

- **特征**：罕见的极端事件，无法用概率估计
- **管理方式**：
  - 压力测试和情景分析
  - 检查所有可预见的风险情景
  - 确保有应对计划
- **心态**：这类事件必须能生存下来

**区域 3：>10 倍 VaR（生存威胁）**

- **特征**：灾难性事件，超出日常经验
- **管理方式**：
  - 对冲或保险
  - 改变业务模式
  - 增加 VaR 额度
- **心态**：可预见事件不应导致这种损失

**管理哲学**

> 一个无法将 3 倍 VaR 损失作为常规事件处理的机构，可能无法活得足够长来建立 VaR 系统。

## CVaR（条件风险价值）

### 定义

**条件风险价值 (CVaR)**，又称**预期尾部损失 (Expected Shortfall, ES)**：损失超过 VaR 阈值后的平均损失。

**公式**：
```
CVaR = E[损失 | 损失 > VaR]
```

**意义**：回答"一旦发生最坏情况，平均损失是多少"。

### 与 VaR 的区别

| 维度 | VaR | CVaR |
|------|-----|------|
| 含义 | 最大可能损失 | 尾部平均损失 |
| 信息量 | 分位点 | 尾部完整信息 |
| 连贯性 | 非连贯风险度量 | 连贯风险度量 |
| 监管应用 | 旧巴塞尔协议 | 新巴塞尔协议（推荐） |

**示例**：
- 95% 置信度下，VaR = 100 万（5%最坏情况下的损失起点）
- CVaR = 150 万（5%最坏情况下的平均损失）

### 为什么 CVaR 更优

1. **连贯性**：满足次可加性，分散化一定降低风险
2. **尾部风险敏感**：不只看分位点，还看尾部形状
3. **监管认可**：巴塞尔委员会 2013 年起推荐用 ES 替代 VaR

## 压力测试

### 定义

**压力测试 (Stress Testing)**：模拟极端但不常见的市场情景，评估投资组合在极端条件下的表现。

**目的**：识别"黑天鹅"事件下的脆弱点，补充 VaR 等统计方法。

### 测试类型

#### 1. 历史情景法

使用历史上真实发生的极端事件：
- 2008 年金融危机
- 2020 年疫情熔断
- 2015 年股灾

**优点**：真实发生过，说服力强

**缺点**：未来危机可能不同于历史

---

#### 2. 假设情景法

设计假设的极端情景：
- 股指下跌 30%
- 利率上升 200bp
- 汇率贬值 20%

**优点**：可定制化，覆盖潜在风险

**缺点**：主观性强

---

#### 3. 反向压力测试

从"组合崩溃"反推：
- 假设组合损失 50%
- 反推什么条件下会发生
- 评估该条件发生的可能性

### 与 VaR 的关系

| 维度 | VaR | 压力测试 |
|------|-----|---------|
| 关注点 | 正常情况 | 极端情况 |
| 概率 | 高置信度（95%-99%） | 低概率事件 |
| 方法 | 统计推断 | 情景模拟 |
| 作用 | 日常风险管理 | 极端风险预警 |

**互补使用**：VaR 管日常风险，压力测试管尾部风险。

---

## 应用场景

### 机构投资者

- 银行资本充足率计算（巴塞尔协议）
- 资产配置决策
- 风险限额设定

### 个人投资者

- 评估组合最大可能损失
- 确定合适的仓位
- 极端情况心理准备

### 实践建议

1. **置信度选择**：日常用 95%，风控用 99%
2. **持有期选择**：与流动性匹配，股票通常 1 天
3. **回溯检验**：定期检验 VaR 预测是否准确
4. **多方法交叉**：同时使用 VaR + CVaR + 压力测试

---

## 注意事项

### VaR 的陷阱

1. **假安全感**：VaR 不是最大损失，只是高置信度下的损失上限
2. **模型依赖**：错误的分布假设导致严重低估风险
3. **忽视相关性**：危机时相关性趋近 1，分散化失效

### 压力测试的陷阱

1. **情景过轻**：测试不够极端，低估真实风险
2. **情景过重**：测试过于极端，失去指导意义
3. **忽视二阶效应**：流动性枯竭、交易对手违约等连锁反应

## 实现要点

### 参数法

**VaR 计算**：
- 计算收益率均值 μ 和标准差 σ
- 根据置信水平查 Z 值（95%→-1.65，99%→-2.33）
- VaR = μ + Z × σ

**CVaR 计算**：
- CVaR = μ - σ × φ(Z) / (1-α)
- φ(Z) 是标准正态分布的 PDF 值

### 历史模拟法

**VaR 计算**：
- 对历史收益率排序
- 取 (1-置信度) 分位点

**CVaR 计算**：
- 找出所有低于 VaR 的收益率
- 计算这些收益率的平均值

### 蒙特卡洛模拟

**步骤**：
- 拟合收益率分布（如 t 分布捕捉厚尾）
- 随机生成数万个情景
- 取分位点作为 VaR

### 组合 VaR（考虑相关性）

**步骤**：
- 计算各资产收益率的协方差矩阵
- 组合方差 = 权重向量ᵀ × 协方差矩阵 × 权重向量
- 组合标准差 = √组合方差
- VaR = Z × 组合标准差

## 回溯检验（Backtesting）

### 目的

验证 VaR 模型的准确性：预测的损失阈值是否与实际发生的突破次数一致。

### 方法

#### 1. 突破计数法

**原理**：在 95% 置信度下，预期突破次数 = 5% × 观察天数

**判断标准**（250 天观察期）：
- 预期突破：12.5 次
- 可接受范围：6-19 次（Kupiec 检验）
- 突破过多：模型低估风险
- 突破过少：模型过于保守

#### 2. Kupiec 检验

**原理**：似然比检验，检验实际突破率是否显著偏离预期

**步骤**：
- 计算实际突破率 p̂ = 突破次数 / 观察天数
- 计算似然比统计量 LR
- LR 服从 χ²(1) 分布，p < 0.05 表示模型有问题

#### 3. 独立性检验（Christoffersen 检验）

**原理**：检验突破是否独立（不应连续发生）

**步骤**：
- 计算突破状态的转移矩阵（00, 01, 10, 11 次数）
- 检验转移概率是否独立
- p < 0.05 表示突破有聚集性，模型需改进

### 监管要求（巴塞尔协议）

| 区域 | 突破次数（250天） | 资本乘数 |
|------|------------------|---------|
| 绿区 | 0-4 | 3.0 |
| 黄区 | 5-9 | 3.4-3.85 |
| 红区 | 10+ | 4.0 |

## 实践经验

### 方法选择

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 股票组合 | 历史模拟法 | 捕捉厚尾，简单直观 |
| 含期权组合 | 蒙特卡洛 | 处理非线性 |
| 高频交易 | 参数法 | 计算速度快 |
| 初步评估 | 参数法 | 快速估算 |

### 参数调优

1. **历史窗口**：
   - 太短（< 100 天）：噪音大
   - 太长（> 500 天）：旧信息权重过高
   - 推荐：250 天（1 年）

2. **置信水平**：
   - 95%：日常监控
   - 99%：资本计算、风控限额

3. **持有期**：
   - 日度：股票、期货
   - 周度/月度：流动性差的资产

### 常见错误

1. **忽视时间缩放**：
   - T 天 VaR ≠ 1 天 VaR × √T（收益率非独立）
   - 正确做法：用 T 天收益率计算

2. **忽略流动性风险**：
   - VaR 假设可以立即平仓
   - 实际：跌停时无法卖出

3. **过度依赖单一指标**：
   - VaR + CVaR + 压力测试 组合使用
   - 定期回溯检验

## 高级概念：VaR 家族的关系

### 风险度量层次

对于金融损失 X（损失为正），有如下不等式关系：

```
VaR₁₋α(X) ≤ RVaRα,β(X) ≤ CVaR₁₋α(X) ≤ EVaR₁₋α(X)
```

**含义**：从左到右，风险度量越来越保守（估计的损失越来越大）。

### 各指标定义

#### 1. VaR (Value at Risk)

**定义**：损失分布的 (1-α) 分位数

```
VaR₁₋α(X) = inf{t ∈ R: P(X ≤ t) ≥ 1-α}
```

**特点**：
- 最基础的风险度量
- 不连贯（非次可加）
- 鲁棒统计量（对异常值不敏感）

#### 2. CVaR (Conditional VaR / Expected Shortfall)

**定义**：损失超过 VaR 阈值后的平均损失

```
CVaR₁₋α(X) = (1/α) ∫₀^α VaR₁₋γ(X) dγ
```

**特点**：
- 连贯风险度量
- 考虑尾部形状
- 巴塞尔协议推荐使用

#### 3. RVaR (Range Value at Risk)

**定义**：VaR 在区间 [α, β] 上的平均值

```
RVaRα,β(X) = (1/(β-α)) ∫α^β VaR₁₋γ(X) dγ
```

**特点**：
- CVaR 的鲁棒版本
- 对异常值不那么敏感
- 介于 VaR 和 CVaR 之间

#### 4. EVaR (Entropic Value at Risk)

**定义**：基于熵概念的风险度量

```
EVaR₁₋α(X) = inf{z⁻¹ ln(MX(z)/α) : z > 0}
```

其中 MX(z) 是损失 X 的矩母函数。

**特点**：
- 最保守的度量
- 信息论解释
- 计算较复杂

### 选择建议

| 指标 | 适用场景 | 原因 |
|------|---------|------|
| VaR | 日常监控、监管报告 | 简单直观，广泛接受 |
| CVaR | 风险预算、组合优化 | 连贯性好，尾部敏感 |
| RVaR | 鲁棒性要求高的场景 | 平衡灵敏度和稳定性 |
| EVaR | 学术研究、极端风控 | 最保守，理论基础强 |
