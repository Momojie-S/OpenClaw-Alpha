# 回测陷阱与偏差

## 定义

量化回测中常见的系统性错误，会导致回测结果与实盘表现严重偏离。理解并规避这些偏差是构建可靠策略的前提。

---

## 核心偏差类型

### 1. 前视偏差（Look-ahead Bias）

**定义**：使用了回测时点无法获得的信息。

**常见表现**：
- 使用当日收盘价做当日交易决策
- 使用财务报告发布日而非实际公告日
- 使用调整后的历史价格（已反映后续分红拆股）
- 使用全量数据计算指标（如用全局最大最小值归一化）

**规避方法**：
- 严格按时间顺序模拟，只用 t-1 及之前数据
- 财务数据使用"实际公告日"而非"报告期"
- 指标计算只用历史窗口内的数据

**示例**：
```
❌ 错误：用今天收盘价判断今天是否买入
✅ 正确：用昨天收盘价判断，今天开盘买入
```

---

### 2. 幸存者偏差（Survivorship Bias）

**定义**：只使用当前仍存续的股票数据，忽略了已退市股票。

**影响**：
- 回测收益率系统性高估
- 低估策略风险（退市股往往跌幅巨大）
- 对小盘股策略影响尤甚

**规避方法**：
- 使用包含退市股的全量数据库（如 CRSP、A股退市股数据）
- 回测时包含所有历史上存在的股票
- 敏感性分析：对比有无退市股的结果差异

**A股特殊性**：
- 退市数量近年增加
- ST、退市整理期需特别处理
- 借壳上市可能导致数据断裂

---

### 3. 过拟合（Overfitting）

**定义**：策略过度适应历史数据的噪音，而非真实规律。

**常见表现**：
- 过多参数（>5个）
- 参数值过于精细（如 MA=17 而非 20）
- 只在特定时期有效
- 样本内表现极佳，样本外崩塌

**检测方法**：
- **样本外测试**：保留 30% 数据不参与参数优化
- **Walk-Forward 分析**：滚动窗口优化和测试
- **参数稳定性**：参数小幅变动不应导致绩效剧变
- **蒙特卡洛模拟**：随机打乱数据测试策略鲁棒性

**规避原则**：
- 参数越少越好
- 使用"整"数（如 20 日而非 17 日）
- 样本内外表现差距 < 30%
- 多市场、多周期验证

---

### 4. 数据窥探偏差（Data Snooping Bias）

**定义**：多次使用同一数据集测试不同策略，偶然找到"有效"策略。

**本质**：
- 假设显著性水平 α=0.05
- 测试 100 个无效策略，期望有 5 个"显著有效"
- 这是伪相关，不是真实规律

**规避方法**：
- **Bonferroni 校正**：显著性阈值 = α / 测试次数
- **样本外验证**：发现策略后用新数据验证
- **预先注册**：交易前明确策略逻辑，而非事后解释
- **经济逻辑检验**：策略是否有合理的经济学解释

---

### 5. 交易成本低估

**类型**：

| 成本类型 | 说明 | 常见低估 |
|---------|------|---------|
| 佣金 | 券商手续费 | 可忽略（现在很低） |
| 印花税 | 卖出 0.1%（A股） | 常被忽略 |
| 冲击成本 | 大单对价格的冲击 | 严重低估 |
| 滑点 | 下单到成交的价格偏移 | 严重低估 |
| 买卖价差 | 委托价与成交价差异 | 中等低估 |

**实战影响**：
- 高频策略：交易成本可能吞噬全部收益
- 小盘股策略：冲击成本巨大
- 趋势策略：滑点可能导致信号失效

**估算方法**：
```
总成本 = 佣金 + 印花税 + 冲击成本 + 滑点
保守估计：单边 0.2%-0.5%（视策略频率和规模）
```

---

### 6. 流动性约束

**定义**：回测假设可以任意数量成交，实盘却受限于流动性。

**常见表现**：
- 涨跌停无法买入/卖出
- 日成交量不足，无法完成预期交易
- 集合竞价价格不确定

**处理方法**：
- **成交量限制**：单日买入量 < 日成交量的 X%
-**涨跌停过滤**：涨跌停日不交易
- **流动性筛选**：只交易流动性充足的股票

## 偏差影响对比

| 偏差类型 | 对收益影响 | 对风险影响 | 严重程度 |
|---------|-----------|-----------|---------|
| 前视偏差 | 高估 | 低估 | ⭐⭐⭐⭐⭐ |
| 幸存者偏差 | 高估 | 低估 | ⭐⭐⭐⭐ |
| 过拟合 | 高估样本内 | 低估样本外 | ⭐⭐⭐⭐⭐ |
| 数据窥探 | 虚假有效 | 低估 | ⭐⭐⭐⭐ |
| 交易成本 | 高估净收益 | 不影响 | ⭐⭐⭐ |
| 流动性约束 | 高估 | 低估 | ⭐⭐⭐ |

## 回测质量检验清单

### 数据层面
- [ ] 是否包含退市股？
- [ ] 财务数据是否使用公告日？
- [ ] 价格是否经过正确复权？
- [ ] 是否有数据缺失？如何处理？

### 逻辑层面
- [ ] 是否有前视偏差？
- [ ] 参数数量是否过多？
- [ ] 是否有经济学解释？
- [ ] 样本内外表现是否一致？

### 交易层面
- [ ] 交易成本如何假设？
- [ ] 是否考虑滑点？
- [ ] 是否考虑涨跌停？
- [ ] 是否考虑流动性约束？

### 统计层面
- [ ] 是否有足够样本（>100 交易）？
- [ ] 收益分布是否正态？
- [ ] 是否测试过参数稳定性？
- [ ] 是否进行 Walk-Forward 验证？

## 实战建议

### 保守原则

**收益打折**：
- 回测年化收益 × 0.5-0.7 作为预期收益
- 高频策略打折更多

**风险加成**：
- 回测最大回撤 × 1.5 作为预期风险
- 加入极端场景测试（如 2015、2020）

### 参数选择

**稳定性优先**：
- 选择参数平原区而非最优点
- 参数小幅变动绩效不变

**经济逻辑**：
- 每个参数要有经济学含义
- 避免数据挖掘式的参数搜索

### 多重验证

1. **样本外验证**：保留数据不参与开发
2. **跨市场验证**：在其他市场测试
3. **跨周期验证**：在不同时间段测试
4. **蒙特卡洛验证**：随机数据鲁棒性测试
