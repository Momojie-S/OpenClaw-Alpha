# 技术指标参数优化

## 定义

通过科学方法寻找技术指标的最佳参数组合，在保持稳健性的前提下提高信号有效性的过程。

---

## 核心原则

### 避免过拟合

**过拟合现象**：参数在历史数据表现完美，实盘完全失效

**原因**：
- 历史数据噪音被当作规律
- 参数空间太大，随机找到"最优"
- 市场结构变化

**防范措施**：
| 方法 | 说明 |
|------|------|
| 样本外测试 | 用 70% 数据优化，30% 数据验证 |
| 参数稳定性 | 微小参数变化不影响表现 |
| 多品种验证 | 参数在多只股票上有效 |
| 逻辑支撑 | 参数有金融逻辑，非纯数据拟合 |

### 简约原则

**Occam's Razor**：参数越少越好，越简单越稳健

**实践**：
- 优先使用默认参数
- 只在有明显改进时才调整
- 避免使用"神奇数字"

## 各指标参数优化

### 均线系统

#### 单均线

**默认**：MA20（月线）

**优化方向**：
| 交易风格 | 推荐周期 | 说明 |
|---------|---------|------|
| 短线 | MA5-MA10 | 敏感，噪音多 |
| 中线 | MA20-MA60 | 平衡 |
| 长线 | MA120-MA250 | 稳定，滞后 |

**检验方法**：
- 比较不同周期均线的穿透次数
- 选择穿透次数最少的周期（支撑/压力最有效）

#### 双均线

**默认**：MA5/MA20 或 MA10/MA60

**优化参数**：
```
短期均线：5-20
长期均线：20-120
```

**优化目标**：
- 最大化金叉/死叉的盈亏比
- 最小化震荡市的假信号

**A 股实证**：
- 5/20：短线敏感，假信号多
- 10/60：中线稳定，推荐
- 20/120：长线可靠，滞后明显

### MACD

**默认参数**：12, 26, 9

**参数含义**：
| 参数 | 默认值 | 含义 |
|------|--------|------|
| 快线周期 | 12 | 短期 EMA |
| 慢线周期 | 26 | 长期 EMA |
| 信号线 | 9 | DEA 的 EMA 周期 |

**常见调整**：

| 场景 | 参数调整 | 效果 |
|------|---------|------|
| 短线 | 6, 13, 5 | 更敏感，假信号增加 |
| 长线 | 19, 39, 9 | 更稳定，信号滞后 |
| 周线 | 改用默认 | 周线本身已降低噪音 |

**不建议过度调整**：
- MACD 的默认参数经过长期验证
- 参数间有数学关系（快/慢≈0.5）
- 调整后需重新理解信号含义

---

### RSI

**默认参数**：14

**参数敏感性分析**：
| 周期 | 特点 | 适用场景 |
|------|------|---------|
| 6-9 | 极敏感，频繁超买超卖 | 短线高频交易 |
| 14 | 平衡敏感度和稳定性 | 通用（推荐） |
| 21-28 | 稳定，信号少 | 长线趋势跟踪 |

**超买超卖阈值**：
| 阈值设置 | 信号频率 | 假信号率 |
|---------|---------|---------|
| 30/70 | 标准 | 中等 |
| 20/80 | 严格 | 低 |
| 40/60 | 宽松 | 高 |

**A 股建议**：
- 日线用 14 周期 + 30/70 阈值
- 熊市可调整为 20/80（减少抄底被套）

---

### KDJ

**默认参数**：9, 3, 3

**参数含义**：
| 参数 | 含义 |
|------|------|
| N（9） | 计算 RSV 的周期 |
| M1（3） | K 值平滑周期 |
| M2（3） | D 值平滑周期 |

**优化方向**：
| 参数组合 | 特点 |
|---------|------|
| 9, 3, 3 | 标准，适合日线 |
| 18, 3, 3 | 稳定，适合周线 |
| 5, 3, 3 | 敏感，适合短线 |

**KDJ 的固有问题**：
- 容易在超买超卖区钝化
- 震荡市效果好，趋势市失效
- 建议与趋势指标组合使用

---

### 布林带

**默认参数**：20, 2

**参数含义**：
| 参数 | 含义 |
|------|------|
| N（20） | 中轨均线周期 |
| K（2） | 标准差倍数 |

**优化方向**：

**中轨周期**：
| 周期 | 说明 |
|------|------|
| 10 | 短线，带宽变化快 |
| 20 | 标准，覆盖月度波动 |
| 50 | 长线，带宽稳定 |

**标准差倍数**：
| 倍数 | 覆盖概率 | 适用 |
|------|---------|------|
| 1.5 | ~86% | 宽松，触及频繁 |
| 2.0 | ~95% | 标准（推荐） |
| 2.5 | ~99% | 严格，触及少 |

**带宽参数**：
- 带宽收窄 → 波动率下降 → 可能突破
- 带宽放大 → 波动率上升 → 趋势加速

## 参数优化方法

### 网格搜索

**步骤**：
1. 定义参数范围
2. 遍历所有参数组合
3. 计算每个组合的绩效指标
4. 选择最优参数

**示例**：双均线优化
```python
short_range = range(5, 21, 5)  # 5, 10, 15, 20
long_range = range(20, 121, 20)  # 20, 40, 60, 80, 100, 120

best_sharpe = -inf
best_params = None

for short in short_range:
    for long in long_range:
        if short >= long:
            continue
        sharpe = backtest(short, long)
        if sharpe > best_sharpe:
            best_sharpe = sharpe
            best_params = (short, long)
```

**缺点**：计算量大，易过拟合

---

### Walk-Forward 分析

**原理**：滚动窗口优化，模拟实盘

**步骤**：
1. 固定回看窗口（如 250 天）
2. 在窗口内优化参数
3. 用最优参数在下一个窗口交易
4. 滚动重复

**优点**：
- 避免前视偏差
- 更接近实盘表现
- 检验参数稳定性

**示例**：
```
窗口1（D1-D250）：优化 → 参数A → 用于 D251-D260
窗口2（D11-D260）：优化 → 参数B → 用于 D261-D270
...
统计所有窗口的实际收益
```

---

### 参数稳定性检验

**方法**：检查最优参数附近的表现

**稳定参数**：
```
MA10/MA60 Sharpe: 1.2
MA11/MA60 Sharpe: 1.18  ← 微小变化，表现相近
MA9/MA60 Sharpe: 1.19
```
→ 稳定，可用

**不稳定参数**：
```
MA10/MA60 Sharpe: 1.2
MA11/MA60 Sharpe: 0.8  ← 微小变化，表现剧变
MA9/MA60 Sharpe: 0.6
```
→ 不稳定，过拟合风险高

## A 股参数建议

### 日线级别

| 指标 | 推荐参数 | 备注 |
|------|---------|------|
| 均线 | 10/60 | 中线趋势 |
| MACD | 12, 26, 9 | 默认即可 |
| RSI | 14, 30/70 | 标准 |
| KDJ | 9, 3, 3 | 需配合趋势指标 |
| 布林带 | 20, 2 | 标准 |

### 周线级别

| 指标 | 推荐参数 | 备注 |
|------|---------|------|
| 均线 | 20/60 | 覆盖季度 |
| MACD | 默认 | 周线本身已稳定 |
| RSI | 14 | 标准 |

### A 股特殊性

**波动率大**：
- 可适当放宽超买超卖阈值（20/80）
- 布林带标准差倍数可调至 2.5

**散户多**：
- 反转效应强
- KDJ 超买超卖信号可参考

**政策敏感**：
- 大事件前减少参数敏感度
- 保持稳定参数，避免过度调整

## 常见误区

### 1. 追求"完美参数"

**问题**：试图找到历史表现最好的参数
**事实**：不存在"完美参数"，只有"稳健参数"

### 2. 频繁调整参数

**问题**：根据近期表现不断调整参数
**后果**：追涨杀跌，总是滞后

### 3. 忽视交易成本

**问题**：优化时只看收益，忽视换手成本
**后果**：实盘被成本侵蚀

### 4. 样本不足

**问题**：用 1 年数据优化
**风险**：样本太少，统计不可靠
**建议**：至少 3-5 年数据
