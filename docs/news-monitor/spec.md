# 财经新闻监控 - 需求文档

## 一、背景

现有的投资分析体系主要依赖数值型数据（行情、资金流、财务指标），缺少文本型信息（新闻、公告、研报）。新闻是影响市场情绪和个股走势的重要因素，需要补充新闻获取和分析能力。

## 二、目标

为 OpenClaw agent 提供财经新闻获取能力，帮助使用者了解市场热点和个股相关新闻。

## 三、用户场景

### 场景 1：懒人查看今日热点

```
使用者：最近有什么财经新闻？
Agent：自动获取今日热点新闻，展示 Top 10
```

**要求**：不传任何参数，自动返回当天最重要的财经新闻。

### 场景 2：关注特定话题

```
使用者：AI 相关的新闻有哪些？
Agent：筛选包含 "AI" 关键词的新闻
```

**要求**：支持关键词筛选，返回相关新闻。

### 场景 3：追踪个股动态

```
使用者：平安银行最近有什么新闻？
Agent：筛选标题或内容包含 "平安银行" 的新闻
```

**要求**：支持股票名称/代码筛选。

## 四、功能需求

### 4.1 默认行为（零参数）

当不传任何参数时：

- 获取 **今日** 热点财经新闻
- 按 **热度/重要性** 排序
- 返回 **Top 10** 条
- 来源：主流财经网站（东方财富、同花顺、新浪财经等）

### 4.2 可选参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--keyword` | 关键词筛选 | 无（返回所有） |
| `--stock` | 股票名称或代码 | 无 |
| `--date` | 日期（YYYY-MM-DD） | 今天 |
| `--top-n` | 返回数量 | 10 |
| `--source` | 新闻来源 | 所有来源 |

### 4.3 输出格式

每条新闻包含：

| 字段 | 说明 |
|------|------|
| 标题 | 新闻标题 |
| 来源 | 新闻网站 |
| 时间 | 发布时间 |
| 摘要 | 内容摘要（如可获取） |
| 链接 | 原文链接 |

**控制台输出**：精简列表（标题 + 来源 + 时间）

**文件输出**：完整数据（含摘要、链接）

### 4.4 新闻来源

优先级（按数据质量/稳定性）：

1. 东方财富 - 热点明确，数据结构化
2. 同花顺 - 覆盖面广
3. 新浪财经 - 更新及时

## 五、非功能需求

### 5.1 性能

- 默认查询（Top 10）应在 **10 秒内** 返回
- 支持网络重试（新闻网站可能不稳定）

### 5.2 容错

- 单个来源失败时，尝试其他来源
- 部分字段缺失时，返回可用字段

## 六、验收标准

- [ ] 不传参数时，返回今日热点新闻 Top 10
- [ ] 传入 `--keyword` 时，返回包含关键词的新闻
- [ ] 传入 `--stock` 时，返回与该股票相关的新闻
- [ ] 每条新闻包含：标题、来源、时间、链接
- [ ] 结果保存到文件，供后续分析使用

## 七、不做的功能

- 实时新闻推送（轮询机制）
- 新闻情感分析（NLP 处理）
- 历史新闻归档（数据库存储）
- 研报/公告获取（单独的 skill）
