# 涨停追踪 - 需求文档

## 一、背景

涨停板是 A 股市场的重要特征，反映市场情绪和热点方向。投资者需要追踪：
- 今日涨停股（数量、板块分布、连板统计）
- 连板股（2连板、3连板等高连板）
- 炸板股（涨停后打开，风险信号）
- 昨日涨停今日表现（追涨收益验证）

## 二、用户场景

### 场景 1：快速了解今日涨停情况

> 用户：今天涨停股有哪些？连板股呢？
> Agent：调用涨停追踪，返回涨停列表 + 连板统计

### 场景 2：追踪连板龙头

> 用户：哪些股票 3 连板以上？
> Agent：筛选连板数 >= 3 的股票

### 场景 3：监控炸板风险

> 用户：今天有没有炸板的？
> Agent：返回炸板股池

### 场景 4：验证追涨收益

> 用户：昨天涨停的今天表现怎么样？
> Agent：返回昨日涨停今日表现数据

## 三、业务概念

### 3.1 涨停类型

| 类型 | 说明 |
|------|------|
| 涨停 | 涨幅达到 10%（主板）或 20%（创业板/科创板） |
| 跌停 | 跌幅达到限制 |
| 炸板 | 涨停后打开，未能封住 |
| 连板 | 连续多日涨停 |

### 3.2 连板数

- 1 板：首板（首次涨停）
- 2 板：连板（连续两天涨停）
- 3+ 板：高连板（市场高度关注）

### 3.3 封板强度

- **封板资金**：封住涨停的资金量（越大越稳）
- **炸板次数**：打开又封住的次数（越多越不稳）
- **封板时间**：首次封板时间（越早越强）

## 四、验收标准

- [ ] 能获取今日涨停股列表
- [ ] 能按连板数筛选（如 2 连板以上）
- [ ] 能获取炸板股池
- [ ] 能获取昨日涨停今日表现
- [ ] 输出包含关键指标（封板时间、炸板次数、连板数等）

## 五、范围边界

### 做

- 涨停/跌停/炸板数据获取
- 连板统计
- 按连板数筛选
- 昨日涨停表现追踪

### 不做

- 实时推送（非实时系统）
- 涨停预测
- 复杂技术分析
