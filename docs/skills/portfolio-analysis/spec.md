# 持仓分析 Skill - 需求文档

## 一、背景

投资者往往持有多只股票，需要了解自己投资组合的整体状况：
- 持仓是否过于集中？
- 行业分布是否均衡？
- 当前整体盈亏如何？

本 skill 帮助用户快速分析自己的投资组合结构。

## 二、目标用户

- 有多只股票持仓的个人投资者
- 想了解组合风险敞口的用户
- 需要定期审视持仓结构的用户

## 三、核心功能

### 3.1 持仓输入

**方式一：命令行参数**
```bash
--holdings "000001:1000,600000:500,002475:200"
```

**方式二：JSON 文件**
```json
{
  "holdings": [
    {"code": "000001", "shares": 1000, "cost": 12.5},
    {"code": "600000", "shares": 500, "cost": 8.2}
  ]
}
```

**字段说明**：
- `code`: 股票代码（必填）
- `shares`: 持有股数（必填）
- `cost`: 成本价（选填，用于计算盈亏）

### 3.2 分析维度

| 维度 | 说明 | 指标 |
|------|------|------|
| 持仓结构 | 各股票占比 | 权重分布、Top 5 |
| 行业分布 | 行业集中度 | 行业权重、HHI 指数 |
| 市值分布 | 大中小盘占比 | 市值区间分布 |
| 盈亏分析 | 各股盈亏 | 盈亏金额、盈亏比例 |
| 风险提示 | 集中风险 | 超限提醒 |

### 3.3 风险提示规则

| 风险类型 | 规则 | 级别 |
|---------|------|------|
| 单股集中 | 单股权重 > 30% | 高 |
| 行业集中 | 单行业权重 > 50% | 高 |
| 持仓过少 | 股票数 < 3 | 中 |
| 持仓过多 | 股票数 > 20 | 低 |

### 3.4 输出格式

**控制台输出**（精简）：
```
持仓分析报告
============
总市值：¥125,680.00
总盈亏：+¥8,320.00 (+7.1%)

持仓分布（按市值）：
1. 平安银行(000001) - 35.2% - ¥44,200
2. 中国平安(601318) - 25.1% - ¥31,550
...

行业分布：
- 金融：60.3%
- 科技：25.1%
...

风险提示：
⚠️ 单股集中度较高：平安银行占比 35.2%
```

**文件输出**（完整）：
```json
{
  "summary": {...},
  "holdings": [...],
  "industry_distribution": {...},
  "risk_alerts": [...]
}
```

## 四、验收标准

- [ ] 支持命令行和 JSON 文件两种输入方式
- [ ] 正确计算各股票权重（市值占比）
- [ ] 正确统计行业分布
- [ ] 当成本价提供时，正确计算盈亏
- [ ] 风险提示符合规则
- [ ] 控制台输出精简，文件输出完整

## 五、不做

- 不做历史持仓追踪
- 不做调仓建议
- 不做风险对冲方案
- 不支持基金、债券等其他资产
