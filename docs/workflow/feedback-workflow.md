# 处理用户反馈任务流程

**核心原则：渐进式填写，完成当前阶段后才填写下一阶段的具体任务。**

**入口**：`feedback/` 目录（OpenClaw-Alpha 项目）

---

## 流程阶段

| Step | 内容 | 输出 |
|------|------|------|
| 1. 读取 | 检查未处理的反馈文件 | 反馈内容 |
| 2. 需求分析 | 区分表面需求 vs 真实需求 | 需求分析 |
| 3. 技术评估 | 检查是否符合项目规范 | 技术评估 |
| 4. 处理决策 | 采纳/调整后采纳/不采纳/待讨论 | 决策结论 |
| 5. 归档 | 追加处理记录，移动到 processed/ | - |

---

## 核心原则

- 理解真实需求，而非照搬方案
- 规范优先，用户建议只是参考
- 不在本次对话中立即实现

---
