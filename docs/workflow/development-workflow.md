# 开发任务流程

**核心原则：渐进式填写，完成当前阶段后才填写下一阶段的具体任务。**

---

## 流程阶段

| Phase | 内容 | 输出 |
|-------|------|------|
| 1. 需求 | 定义业务需求，输出分析思路 | 需求文档 |
| 2. 调研 | 根据分析思路选择调研方式 | 技术方案 |
| 3. 文档 | 根据调研结果确定文档类型 | API/设计文档 |
| 4. 开发 | 根据技术方案开发 | 代码/SKILL.md |
| 5. 验证 | 根据开发内容选择验证方式 | 测试结果 |
| 6. 回顾 | 文档合并，总结经验，规划后续 | 更新规范 |
| 7. 提交 | git commit + push | - |

---

## Phase 5: 验证

**根据开发内容选择验证方式**：

| 验证方式 | 何时需要 |
|---------|---------|
| **API 调试** | 使用外部 API（需要调试 API 获取真实结果） |
| **单元测试** | 涉及 Python 脚本 |
| **流程验证** | 编排已有能力 |
| **Browser 测试** | 需要浏览器访问 |

---

## 关键原则

1. **分析思路决定调研方向** - 需求文档的分析思路决定后续用什么方式调研
2. **调研结果决定开发任务** - 调研完成后才知道需要开发什么
3. **开发内容决定验证方式** - 有 API 就要调试，有代码就要测试

---
