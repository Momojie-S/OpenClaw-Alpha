# 学习调研任务流程

**核心原则：渐进式填写，完成当前阶段后才填写下一阶段的具体任务。**

---

## 流程阶段

| Step | 内容 | 输出 |
|------|------|------|
| 1. 查看现有 | 阅读知识体系，确定学习方向 | 学习方向 |
| 2. 深入学习 | Web Search / Web Fetch / 查阅书籍 | 理解与要点 |
| 3. 完善文档 | 编写 Obsidian 风格笔记 | 笔记文件 |
| 4. 反哺框架 | 将新知识应用到分析框架（可选） | 框架更新 |
| 5. 规划后续 | 识别后续学习或开发需求 | 后续任务 |

---

## 能力状态标记

在 `analysis-framework.md` 中标记能力状态时：
- **待开发** - 研究完成，需要后续开发实现
- **已完成** - 已实现并可使用
- **不适用** - 经过研究后确定不需要或无法实现

---

## 进度模板

```markdown
# 学习调研：XXX

## 主题
（要学习的主题）

## Step 1: 查看现有
- [ ] 阅读 README 和已有笔记
- [ ] 确定学习方向

## Step 2: 深入学习
（根据 Step 1 的学习方向填写）
- [ ] 学习任务 1
- [ ] 整理关键要点

## Step 3: 完善文档
- [ ] 编写/更新笔记
- [ ] 添加 [[wikilinks]] 关联

## Step 4: 反哺框架（可选）
- [ ] 审视 analysis-framework.md
- [ ] 更新框架（如有改进）
- [ ] 新能力标记为"待开发"（如有）

## Step 5: 规划后续任务
- [ ] 回顾疑问和未深入方向
- [ ] 检查是否触发新需求

## 状态
- **当前步骤**：（Step 1-5）
- **进度**：（正常/暂停）
- **下一步**：（当前步骤完成后要做什么）

## 后续任务（可选）
- **任务名称**：
- **任务类型**：（开发/调研/学习）
- **触发原因**：
- **优先级**：

## 学习笔记
（关键发现、理解）
```
