# Skill 自检任务流程

**核心原则：渐进式填写，完成当前阶段后才填写下一阶段的具体任务。**

**目的**：从使用者视角真实使用 skill，发现问题并创建改进任务。

---

## 流程阶段

| Step | 内容 | 输出 |
|------|------|------|
| 1. 找真实使用场景 | 根据当前市场情况或投资需求，找到该 skill 的实际使用场景 | 使用场景 + 输入数据 |
| 2. 真实使用 | 像真实用户一样完整使用 skill，记录体验 | 使用过程 + 输出结果 |
| 3. Review 和反馈 | 回顾使用过程，发现问题和改进点 | 反馈 + 任务文件（可选）|
| 4. 更新总记录 | 更新 self-check-log.md | - |

---

## Step 1: 找真实使用场景

**核心原则：从最近新闻中找到适合该 skill 的真实使用场景**

**场景来源**：RSSHub 财经新闻

**方法**：
1. **获取最近新闻** - 使用 RSSHub 获取财经新闻：
   - 36kr：`curl https://rsshub.app/36kr/news`
   - 华尔街见闻：`curl https://rsshub.app/wallstreetcn/news`
   - 金十数据：`curl https://rsshub.app/jin10`

2. **从新闻中找场景** - 阅读新闻，找到适合该 skill 的使用场景：
   - 新闻提到某个板块大涨 → 用 industry_trend 分析该板块热度
   - 新闻提到外资流入 → 用 northbound_flow 查外资动向
   - 新闻提到某公司业绩 → 用 fundamental_analysis 看基本面
   - 新闻提到市场大跌 → 用 market_sentiment 看情绪

3. **提取输入数据** - 从新闻中提取：
   - 股票代码（6位数字）
   - 板块/概念名称
   - 行业名称

**示例场景**：
- 新闻"半导体板块集体大涨" → fundamental_analysis 分析龙头股基本面
- 新闻"北向资金净流入超百亿" → northbound_flow 查外资买了什么
- 新闻"市场恐慌情绪蔓延" → market_sentiment 看情绪指标

---

## Step 2: 真实使用

**核心原则：像真实用户一样使用，不要只跑一条命令**

**执行步骤**：
1. **阅读 SKILL.md** - 像第一次使用的用户一样阅读
   - 文档是否清晰？能否快速理解如何使用？
   - 参数说明是否完整？示例是否足够？

2. **完整使用流程** - 按 SKILL.md 的步骤真实使用
   - 使用真实数据，不用 mock
   - 选择符合场景的参数组合
   - 记录每个步骤的体验和输出

3. **记录使用过程** - 详细记录：
   - 使用了哪些命令/参数
   - 输出结果是什么
   - 有没有报错或异常
   - 体验如何（流畅/卡顿/困惑）

---

## Step 3: Review 和反馈

**核心原则：回顾使用过程，发现问题和改进机会**

根据实际使用情况自由发挥：
- 发现什么问题就记录什么
- 有什么改进建议也记录下来
- 不按固定维度检查

**问题分类**：
| 严重程度 | 定义 | 处理方式 |
|----------|------|----------|
| **高** | 无法运行 / 输出错误数据 | 立即创建任务，优先修复 |
| **中** | 体验问题 / 文档不清 / 输出格式不佳 | 创建任务，等待排队 |
| **低** | 优化建议 / 边缘场景 | 创建任务，等待排队 |

**任务创建**：所有优先级都要创建任务文件，只是处理顺序不同。

## Step 4: 更新总记录

更新 `docs/skills/self-check-log.md`：

- 上次自检时间
- 后续待办
