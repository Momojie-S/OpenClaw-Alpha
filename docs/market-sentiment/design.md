# 市场情绪分析 - 设计文档

## 一、技术选型

### 数据源

| 数据 | 来源 | 接口 | 积分要求 |
|------|------|------|----------|
| 涨停股池 | AKShare | `stock_zt_pool_em` | 免费 |
| 炸板股池 | AKShare | `stock_zt_pool_zbgc_em` | 免费 |
| 大盘资金流向 | AKShare | `stock_market_fund_flow` | 免费 |
| 涨跌停统计 | Tushare | `limit_list_d` | 2000 |
| 涨跌家数 | Tushare | `daily`（统计） | 120 |

**选择理由**：
- AKShare 免费，优先使用
- Tushare `limit_list_d` 需要 2000 积分，作为备选
- 涨跌家数通过 `daily` 接口统计，只需 120 积分

### 数据获取策略

**第一版（MVP）**：
- 使用 AKShare 获取涨停股池
- 通过涨跌幅统计涨跌家数（可能不准确）
- 使用 AKShare 获取大盘资金流向

**后续版本**：
- 积分足够时，使用 Tushare `limit_list_d` 获取完整涨跌停统计

### 类设计

#### LimitFetcher（涨跌停数据）

**职责**：获取涨跌停统计数据

**输入**：
- `date`: 交易日期

**输出**：
```json
{
  "date": "2026-03-06",
  "limit_up": 74,
  "limit_down": 0,
  "break_board": 26  // 炸板数
}
```

#### FlowFetcher（资金流向）

**职责**：获取大盘资金流向数据

**输入**：
- `date`: 交易日期（可选，默认最近）

**输出**：
```json
{
  "date": "2026-03-06",
  "main_net_inflow": -1234567890.12,  // 主力净流入
  "main_net_inflow_pct": -0.85,        // 主力净流入占比
  "retail_net_inflow": 26443550000.00, // 散户净流入（小单）
  "retail_net_inflow_pct": 1.20        // 散户净流入占比
}
```

#### SentimentProcessor（情绪分析）

**职责**：综合计算市场情绪

**输入**：
- `date`: 交易日期

**处理流程**：
1. 调用 LimitFetcher 获取涨跌停数据
2. 调用 FlowFetcher 获取资金流向
3. 调用 Tushare daily 接口统计涨跌家数（或通过其他方式）
4. 计算市场温度评分
5. 判断市场情绪状态
6. 识别极端信号

**输出**：
```json
{
  "date": "2026-03-06",
  "limit": {
    "up": 74,
    "down": 0,
    "break_board": 26
  },
  "trend": {
    "up": 4260,
    "down": 1154,
    "flat": 63
  },
  "flow": {
    "main_net_inflow": -1234567890.12,
    "main_net_inflow_pct": -0.85
  },
  "sentiment": {
    "temperature": 65,       // 市场温度（0-100）
    "status": "偏热",        // 极度恐慌/偏冷/正常/偏热/极度亢奋
    "signals": ["主力流出", "涨停数较多"]
  }
}
```

## 三、评分算法

### 市场温度计算

市场温度由多个维度加权计算：

```
市场温度 = 涨停维度 × 0.3 + 涨跌家数维度 × 0.4 + 资金流向维度 × 0.3
```

#### 涨停维度（0-100 分）

| 涨停数 | 分数 |
|--------|------|
| > 200 | 100 |
| 100-200 | 80 |
| 50-100 | 60 |
| 20-50 | 40 |
| < 20 | 20 |

#### 涨跌家数维度（0-100 分）

```
涨跌家数得分 = (上涨家数 / 总家数) × 100
```

#### 资金流向维度（0-100 分）

| 主力净流入占比 | 分数 |
|----------------|------|
| > 1% | 100 |
| 0.5%-1% | 80 |
| 0-0.5% | 60 |
| -0.5%-0 | 40 |
| < -0.5% | 20 |

### 情绪状态判断

| 温度范围 | 状态 |
|----------|------|
| 80-100 | 极度亢奋 |
| 60-80 | 偏热 |
| 40-60 | 正常 |
| 20-40 | 偏冷 |
| 0-20 | 极度恐慌 |

### 极端信号识别

| 信号 | 触发条件 |
|------|----------|
| 过热预警 | 涨停数 > 200 或 温度 > 90 |
| 恐慌底部 | 跌停数 > 100 或 温度 < 10 |
| 主力流出 | 主力净流入占比 < -1% |
| 分化严重 | 涨停数 > 50 且 跌停数 > 20 |

## 四、接口定义

### AKShare 接口

#### stock_zt_pool_em（涨停股池）

- 输入：`date`（YYYYMMDD）
- 输出：涨停股票列表
- 字段：代码、名称、涨跌幅、封板时间等
- 使用：统计涨停家数

#### stock_zt_pool_zbgc_em（炸板股池）

- 输入：`date`（YYYYMMDD）
- 输出：炸板股票列表
- 使用：统计炸板家数

#### stock_market_fund_flow（大盘资金流向）

- 输入：无（返回历史数据）
- 输出：每日资金流向数据
- 字段：日期、主力净流入、散户净流入等

### Tushare 接口

#### daily（日线行情）

- 输入：`trade_date`（YYYYMMDD）
- 输出：所有股票日行情
- 使用：统计涨跌家数

#### limit_list_d（涨跌停统计）- 需要 2000 积分

- 输入：`trade_date`（YYYYMMDD）
- 输出：涨跌停股票列表
- 字段：涨停/跌停标记（U/D）

## 五、边界处理

### 数据缺失

- 涨停数获取失败 → 使用涨跌幅估算（涨幅 >= 9.9%）
- 跌停数获取失败 → 使用涨跌幅估算（跌幅 <= -9.9%）
- 资金流向获取失败 → 该维度不参与评分

### 非交易日

- 返回最近交易日的数据
- 提示用户"非交易日"

### 网络错误

- 重试机制（3 次）
- 降级方案（使用备用数据源）

## 六、文件输出

### 保存路径

`.openclaw_alpha/market-sentiment/{date}/sentiment.json`

### 文件内容

完整的情绪分析结果，包括：
- 原始数据（涨跌停、涨跌家数、资金流向）
- 计算结果（温度、状态、信号）
- 元数据（更新时间、数据源）
