# 个股对比分析

横向对比多只股票的关键指标，帮助投资决策。

## 使用说明

### 脚本运行

```bash
# 对比 2-10 只股票
uv run --env-file .env python -m openclaw_alpha.skills.stock_compare.compare_processor.compare_processor "000001,600000,002475"

# 保存对比结果
uv run --env-file .env python -m openclaw_alpha.skills.stock_compare.compare_processor.compare_processor "000001,600000" --output

# 输出 JSON 格式
uv run --env-file .env python -m openclaw_alpha.skills.stock_compare.compare_processor.compare_processor "000001,600000" --json
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `symbols` | 股票代码（逗号分隔，2-10 只） | 必填 |
| `--date` | 对比日期 | 今天 |
| `--output` | 保存结果到文件 | False |
| `--json` | 输出 JSON 格式 | False |

### 运行记录

运行记录保存在 `.openclaw_alpha/stock_compare/YYYY-MM-DD/compare_processor.json`

### Step 2: 分析评分

**输入**：Step 1 的输出

**动作**：查看各维度得分

**输出**：评分解读

| 评分维度 | 计算方式 | 满分 |
|---------|---------|:----:|
| 估值得分 | PE 越低越高（<15: 90, <30: 70, <50: 50） | 90 |
| 动量得分 | 涨跌幅贡献（50 + 涨幅*5） | 100 |
| 流动性得分 | 换手率适中最佳（3-8%: 80） | 80 |
| 资金得分 | 主力净流入（需要额外数据） | 100 |

### Step 3: 综合决策

**输入**：对比结果 + 其他分析

**动作**：结合分析框架综合评估

**建议**：
- 用 stock_analysis 深入分析评分最高的股票
- 用 fundamental_analysis 查看财务指标
- 用 industry_trend 查看所在板块热度

## 评分体系

### 权重分布

| 维度 | 权重 | 说明 |
|------|:----:|------|
| 估值得分 | 25% | 低估值有安全边际 |
| 动量得分 | 30% | 趋势是朋友 |
| 流动性得分 | 25% | 便于进出 |
| 资金得分 | 20% | 主力资金方向 |

### 评分细则

**估值得分**（PE）：
- PE < 15：90 分
- 15 ≤ PE < 30：70 分
- 30 ≤ PE < 50：50 分
- PE ≥ 50：30 分

**动量得分**（涨跌幅）：
- 基础分 50
- 每涨 1% 加 5 分
- 范围限制在 0-100

**流动性得分**（换手率）：
- 3% ≤ 换手率 ≤ 8%：80 分
- 换手率 < 3%：50 分
- 换手率 > 8%：递减

## 使用场景

### 场景 1：同行业选股

```
使用者：银行股选哪只？平安银行、浦发银行、招商银行？
Agent：运行 stock_compare 对比三只银行股
```

### 场景 2：备选股筛选

```
使用者：我看中了三只股票，帮我对比一下
Agent：运行 stock_compare，返回综合评分
```

### 场景 3：组合优化

```
使用者：持仓中哪只可以加仓？
Agent：运行 stock_compare 对比持仓股票
```

## 注意事项

1. **评分参考**：评分仅供参考，不构成投资建议
2. **数据延迟**：盘中数据有延迟
3. **对比数量**：最多对比 10 只股票
4. **PE 缺失**：部分股票无 PE 数据时，估值得分为 0
